# C33 · Crunch SQL — Syllabus

**Format:** 12 weeks · self-paced · open-source (GPL-3.0). Each week = 3 lectures, 3 exercises, 2 challenges, a mini-project, homework, and a quiz.

**Prerequisites:** You can run commands in a terminal. No prior database or SQL knowledge assumed — this course goes from zero to expert.

**Assessment (honor-based):** tick each lesson complete in the reader; finish the weekly mini-project and quiz. A certificate is issued on 100% completion to One-Stop members.

| Week | Theme | Mini-project |
|------|-------|--------------|
| 1 | Relational model + SELECT (filter/sort/DISTINCT/LIMIT) | Answer 20 business questions from one table |
| 2 | Joins (inner/outer/self/cross) + UNION/INTERSECT/EXCEPT | Reconstruct a report across 5 joined tables |
| 3 | Aggregation (GROUP BY/HAVING) + subqueries + CTEs | Build a layered analytics query with CTEs |
| 4 | Data modeling + normalization (1NF–BCNF) + constraints | Design + create a normalized schema from a spec |
| 5 | Transactions, ACID, isolation levels, MVCC, locking | Reproduce + fix a concurrency anomaly |
| 6 | Indexing deep-dive (B-tree/hash/GIN/GiST/BRIN, partial/covering) | Cut a slow query with the right index |
| 7 | Query optimization — EXPLAIN/ANALYZE, plans, join algos, stats | Tune a 10s query to under 100ms |
| 8 | Window functions + advanced analytics SQL | Build a cohort/running-total dashboard query |
| 9 | Views, functions, stored procedures (PL/pgSQL), triggers | Ship an audited table with triggers + functions |
| 10 | Security — roles, privileges, row-level security, integrity | Lock a multi-tenant schema with RLS |
| 11 | Partitioning + replication + sharding + scaling; SQL vs NoSQL | Partition + set up a read replica |
| 12 | Capstone — profile + tune a real DB; design a production schema | Design + tune a database to a latency target |

**Outcome:** design, secure, and tune a production database — and read a query planner like a senior engineer.
